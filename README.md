# Public Notes Site

This directory contains a Quartz-based public site for selected notes from your Obsidian vault.

The vault source is configured explicitly in [public-notes-site.config.mjs](/home/eviltwin7648/Documents/Obsidian%20Vault/public-notes-site/public-notes-site.config.mjs), so this site can live outside the vault and still read only from that path.

## Workflow

1. Keep writing in the vault as usual.
2. Add frontmatter to any note you want public:

```yaml
---
publish: true
title: Optional public title
description: Optional summary
---
```

3. From this directory, export and preview:

```bash
npm run dev-public
```

4. For a production build:

```bash
npm run build-public
```

## What gets published

- Only Markdown notes with `publish: true`
- Images and attachments referenced by those public notes
- An autogenerated home page listing all exported notes
- Only files from the configured `vaultRoot`

## Privacy behavior

- Notes without `publish: true` are never exported.
- Links from public notes to private notes are converted to plain text during export.
- The exporter works as an allowlist, not a denylist.

## Before deploying

- Update `pageTitle` and `baseUrl` in `quartz.config.ts`
- Add your own favicon later if you want one
- Push only this site to a public repo, or deploy from this subdirectory with your host of choice
